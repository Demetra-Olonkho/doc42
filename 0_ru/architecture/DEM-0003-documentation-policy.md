# DEM-0003 — Политика ведения документации

Версия: 0.1 (Draft)

Статус: Accepted

---

# 1. Назначение

Настоящий документ определяет правила подготовки, хранения и сопровождения документации проекта Demetra Olonkho.

---

# 2. Исходный язык документации

Исходным языком документации проекта является русский язык.

Все нормативные документы первоначально создаются на русском языке.

К таким документам относятся:

* архитектурные документы;
* ADR;
* RFC;
* спецификации;
* проектные решения;
* глоссарии;
* регламенты.

---

# 3. Английская версия

После принятия документа создаётся его официальная английская версия.

Английская версия является официальным переводом исходного документа.

---

# 4. Приоритет версий

При возникновении расхождений между версиями документа:

* русская версия считается исходной;
* английская версия должна быть приведена в соответствие с русской.

---

# 5. Структура документации

Документация организуется по языковым каталогам.

Базовая структура:

docs/
├── 1_ru/
└── 2_en/

При необходимости могут добавляться дополнительные языки:

docs/
├── 1_ru/
├── 2_en/
├── 3_fr/
├── 4_de/
└── ...

---

# 6. Зеркальность структуры

Структура каталогов различных языков должна быть максимально одинаковой.

Пример:

docs/
├── 1_ru/
│   └── architecture/
│       └── DEM-0001.md
│
└── 2_en/
└── architecture/
└── DEM-0001.md

---

# 7. Цель

Данная политика предназначена для обеспечения:

* сохранности первоначального замысла проекта;
* удобства международного участия;
* единообразия документации;
* долгосрочной сопровождаемости.

Конец документа.
